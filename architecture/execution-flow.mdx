# Execution Flow

This page describes the **core execution flow** from the moment the merchant starts a session to the moment fiat reaches the merchant’s bank account.

## High-level steps

1. **Merchant app** triggers the Bleepay SDK to start a payment session (amount, currency, IBAN, webhook, order_id).
2. **User** generates a 6-digit session code in their non-custodial wallet, creating a secure link between their assets and the merchant’s intent.
3. **Merchant** redeems the code via the SDK; Bleepay synchronizes transaction metadata with the backend.
4. **SDK/backend** generates the specific transaction calldata (swap/bridge/transfer as needed) and requests the user’s signature.
5. **User** signs in their wallet; the wallet broadcasts the transaction to the liquidation partner’s infrastructure (or the appropriate chain).
6. **Liquidation partner** performs crypto-to-fiat conversion and sends funds to the merchant’s IBAN.
7. **Bleepay** sends a webhook to the merchant so they can finalize the order.

## Key technical points

- **Orchestration only**: Bleepay coordinates the flow and builds transaction payloads; it does not custody assets or sign on behalf of the user.
- **Client-side signing**: All cryptographic signatures are produced in the user’s wallet.
- **Settlement**: Actual value movement is on-chain and/or via the liquidation partner; Bleepay does not sit in the middle as a custodian.
